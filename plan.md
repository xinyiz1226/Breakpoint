# Breakpoint: Tennis Highlight Extraction Plan

输入：网球比赛电视转播视频（无比分板、无解说、无观众，纯现场原声）

输出：自动剪辑的精彩片段合集

---

## Phase 1 — Audio-Based Point Segmentation & Ranking (MVP)

目标：仅通过音频信号，将比赛切分为逐分片段并排序出精彩回合。

### 1.1 击球音检测 (Hit Detection)
- 使用 onset detection（短时能量突变 / spectral flux）检测击球瞬间
- **分窗检测**：将音频按 60s 窗口切分，每个窗口独立做 onset detection，避免全局阈值漏检较安静片段的击球
- 频段过滤：击球声集中在低中频，与鞋底摩擦声（高频）、球落地弹跳声区分
- 工具：`librosa.onset.onset_detect` + 带通滤波 200-4000Hz + delta=0.2

### 1.2 分段 (Point Segmentation)
- 检测击球之间的长静音间隔（>6秒），作为每一分的分界
- **最大时长限制**：超过 25s 的片段在最大内部间隔处递归拆分（实际回合不超过此时长）
- **最小时长过滤**：丢弃 <4s 的片段（发球失误等非精彩片段）
- 输出每一分的起止时间戳列表

### 1.3 特征提取
- 每一分计算特征：
  - 回合拍数（击球次数）
  - 击球节奏（平均击球间隔时间）
  - 峰值击球强度（最大能量值）
  - 节奏变化（击球间隔的方差，体现攻防转换）

### 1.4 输出
- 导出所有有效片段（不限数量），按原片时间戳顺序排列
- 时间戳列表 + 评分（JSON）
- 使用 `ffmpeg` 裁剪对应视频片段（`-c copy` 不重编码）

## Phase 2 — Vision-Based Feature Enhancement

目标：通过视觉分析增强精彩程度判断。

### 2.1 球轨迹估计
- 帧差分或轻量目标检测追踪球的运动轨迹
- 估算球速变化

### 2.2 球员运动强度
- 光流法计算球员区域运动幅度
- 检测大幅度移动（救球、飞扑）

### 2.3 特殊击球识别
- 上网（球员位置前移）
- 放小球（球速骤降）
- 鱼跃救球（姿态剧变）

### 2.4 融合
- 将视觉特征与 Phase 1 的音频特征合并

---

## Phase 3 — Rally Structure & Smart Compilation

目标：智能组合精彩片段，输出成品视频。

### 3.1 综合评分
- 音频 + 视觉特征统一打分

### 3.2 多样性选择
- 避免选取多个相似回合（如连续5个长回合）
- 兼顾不同类型的精彩（长回合、暴力击球、精彩救球等）

### 3.3 转场 & 后期
- 片段间淡入淡出
- 关键击球慢动作回放（可选）

### 3.4 导出
- 合并为最终精彩集锦视频，保留现场原声

## Tech Stack

- Python 3.10+
- `librosa` — 音频分析
- `ffmpeg` / `ffmpeg-python` — 音视频处理
- `opencv-python` — 视觉分析（Phase 2）
- `numpy`, `scipy` — 数值计算

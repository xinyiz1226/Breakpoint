# English Desktop App Design

## Goal

Create a bilingual desktop app experience for Breakpoint. The app should choose a sensible default language from the user's system language, let users switch between English and Chinese at any time, and remember that choice for future sessions.

The first implementation scope is the desktop app UI only. It does not include separate installers, app-store metadata, website copy, or documentation screenshot updates.

## Target audience and tone

The English experience targets general tennis consumers and enthusiasts. Copy should be easy to understand, slightly marketing-friendly, and focused on the user's outcome: finding rallies, reviewing highlights, and exporting a clean highlight reel.

Chinese copy should preserve the current product feel unless a string must change to match the new localization structure.

## Recommended approach

Use a lightweight in-repo i18n layer instead of introducing a full i18n library.

Add `desktop/src/renderer/i18n/` with:

- Typed English and Chinese copy dictionaries.
- A language type such as `Language = 'en' | 'zh'`.
- A `LanguageProvider` that owns the current language, system default detection, user switching, and persistence.
- Hooks such as `useCopy()` and `useLanguage()` for components.

This fits the current React/Electron app because most copy is local to renderer components, the app has simple interpolation needs, and there is no current requirement for plural rules, remote translations, or many locales.

## Architecture

UI components should not hard-code user-facing strings. Components read copy through the localization hooks and pass the required copy or language into view-model helpers.

The initial conversion should cover these surfaces:

- `WelcomeScreen`: hero tagline, description, import actions, drag-and-drop hint, recent video label, keyboard shortcut labels, and language switch.
- `AnalysisScreen` and `AnalysisProgressPanel`: running state, failed state, stage labels, progress labels, retry/cancel/return actions.
- `MatchMap`: title, subtitle, legend labels, filter toggle, tooltips where practical.
- `RallyQueue`: queue title, summary, batch actions, empty state, export panel, trim labels, tone labels, and open/cancel export actions.
- `App`: resource error wrapper, review-page title, top navigation actions, export success/error message prefixes.
- `flowCopy.ts`: dynamic analysis text, review summary, export button text, segment tone/title helpers.

Business logic should stay language-agnostic. App state, reducers, video analysis, export logic, and Electron main/preload APIs should not depend on a selected language unless they already render user-visible text.

## Language selection

On first launch, the app should derive the default language from the runtime locale:

- `zh-*` locales default to Chinese.
- Other locales default to English.

If a user has already chosen a language, the saved preference takes priority over system language. Switching language updates the current UI immediately and persists the preference for future launches.

The main switch should be visible on the welcome screen. A compact switch should remain available after a video is opened, so users can recover if they picked the wrong language.

Preference storage failures should not block the app. If reading the saved preference fails, fall back to the system-derived default. If writing the preference fails, keep the current in-memory selection active for the current session.

## Copy and interpolation

Keep interpolation small and explicit. Dynamic strings such as export status, segment progress, selected rally counts, and rally titles should be generated by helper functions that receive the active copy or language.

Examples:

- "Export selected rallies" / "导出已选择的回合"
- "Exporting 3 rallies" / "正在导出 3 个回合"
- "Segment 1 / 32 · 25%" / "片段 1 / 32 · 当前 25%"

User-provided values such as paths, file names, ffmpeg errors, and analysis error details should not be translated. Translate the surrounding label or prefix only.

## Error handling

Missing translation keys should be caught during development through shared TypeScript types for both dictionaries. The implementation should avoid stringly typed lookup paths that can silently return blank UI.

Runtime errors from analysis and export should continue to surface clearly. The localized UI should translate action labels and user-facing prefixes, while preserving technical details that help diagnose failures.

## Testing

Update the existing renderer flow tests to cover both languages for the core copy helpers and UI source checks.

At minimum, tests should cover:

- System/default language selection behavior.
- Manual language switching and preference persistence behavior at the provider/helper level.
- Analysis progress copy in English and Chinese.
- Export action copy in English and Chinese.
- Rally title/tone copy in English and Chinese.
- The expectation that English mode core screens do not render Chinese UI strings on the main welcome, analysis, and review paths.

Existing Chinese behavior should remain covered so localization does not accidentally regress the current app.

## Release boundary

Do not create separate Chinese and English installers in this iteration. The shipped desktop app should contain both languages and choose the first-run default automatically.

Do not update the website, docs screenshots, or public marketing copy as part of the first desktop localization implementation. Those can follow after the in-app language model is stable.

## Completion criteria

The work is complete when:

- Users can switch between English and Chinese from the desktop app.
- First-run language defaults to Chinese for `zh-*` system locales and English otherwise.
- The selected language is remembered across sessions.
- Major desktop app user flows have localized copy: welcome/import, analysis progress/error, review queue, match map, trimming, and export.
- English mode no longer shows Chinese UI copy on the main desktop paths.
- Chinese mode preserves the current user experience.
- Existing build and renderer flow tests pass after localization updates.
